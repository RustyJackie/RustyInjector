# Code of Conduct

Short version: be reasonable, stay on topic, don't be a dick.

## What this project is

A tool for authorized security research, penetration testing, and learning how
Linux process injection works at a low level. The target audience is people who
understand what ptrace(2) is and why they'd want to use it.

## Expected behavior

- Discuss technical topics technically — back up claims with facts
- Give constructive feedback on PRs and issues
- If something is broken, say how to reproduce it
- Respect that maintainers do this in their spare time

## What will get you removed

- Asking for help using this on systems you don't own
- Sharing payloads or techniques specifically designed to cause harm
- Harassment of any kind

## Scope

This applies to the GitHub repo — issues, PRs, and discussions. What you do
elsewhere is your business.
